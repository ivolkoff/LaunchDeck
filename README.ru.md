# LaunchDeck

[English](README.md) | **Русский**

Терминальный интерфейс (TUI) для `launchctl` на macOS: просмотр GUI- и
системных сервисов в боковом меню, инспекция метаданных / логов / сырого вывода
`print`, запуск действий start / restart / stop / enable / disable / load /
unload — с поддержкой мыши и восстановлением сессии.

Один статический Go-бинарь, без фонового демона.

![Демо LaunchDeck](docs/demo.gif)

## Установка

С Go 1.24+ на macOS:

```bash
go install github.com/ivolkoff/launchdeck/cmd/launchdeck@latest
```

Либо скачайте готовый бинарь под macOS (Intel `amd64` или Apple Silicon
`arm64`) со страницы [Releases](https://github.com/ivolkoff/launchdeck/releases),
распакуйте архив и положите `launchdeck` в `PATH`.

## Сборка

Из клона репозитория:

```bash
go build -o launchdeck ./cmd/launchdeck
```

Требуется macOS (`launchctl` в `PATH`) и Go 1.24+.

## Запуск

```bash
./launchdeck
```

Состояние сессии (выбор, фильтр, скоуп домена, сортировка, прокрутка списка,
активная вкладка, ширина бокового меню) сохраняется в
`~/.config/launchdeck/session.json` и восстанавливается при следующем запуске.
Вкладка **Логи** обновляется вживую (перечитывает хвост каждые ~2с, новые сверху).

## Цвета (тема)

Цвета настраиваются через `~/.config/launchdeck/theme.json`. Файл
опционален — любое пропущенное поле берётся из встроенной палитры, а отсутствующий
или битый файл использует все значения по умолчанию. Каждое значение — цвет,
понятный lipgloss (индекс 256-цветовой палитры вроде `"42"` или hex вроде
`"#8be9fd"`):

```json
{
  "border": "240",
  "selected_fg": "231",
  "selected_bg": "62",
  "running": "42",
  "stopped": "244",
  "gone": "203",
  "tab_active_fg": "231",
  "tab_active_bg": "62",
  "gutter_fg": "250",
  "gutter_bg": "238",
  "accent": "213",
  "muted": "244"
}
```

## Язык

Интерфейс доступен на английском и русском. Язык выбирается при старте:
`~/.config/launchdeck/config.json` имеет приоритет, если задан; иначе язык
определяется автоматически из `$LC_ALL` / `$LANG` / `$LANGUAGE` (локаль `ru*`
выбирает русский, всё остальное — английский).

```json
{
  "lang": "ru"
}
```

`lang` принимает `"ru"` или `"en"`. Уберите файл (или поле) для автоопределения.
Вывод CLI `--help`/`--version` остаётся на английском.

## Клавиши

Пока открыт модал (поле фильтра, запрос загрузки, action picker, подтверждение,
sudo-повтор), активны только его клавиши; иначе все клавиши ниже глобальны.

| Клавиша                   | Действие                                            |
|---------------------------|------------------------------------------------------|
| `↑`/`k`, `↓`/`j`          | фокус=меню: перемещение выбора (список прокручивается за ним); фокус=детали: прокрутка Логов/Raw |
| `Home` / `End`            | первая / последняя видимая строка                   |
| `PgUp` / `PgDn`           | выбор ±одна страница (10 строк)                      |
| `Tab`                     | переключить фокус меню ↔ детали                      |
| `1` / `2` / `3`           | вкладка деталей: Метаданные / Логи / Raw            |
| `←` / `→`                 | циклом по вкладкам деталей (назад/вперёд)            |
| `Ctrl-U` / `Ctrl-D`      | прокрутка тела Логов/Raw в панели деталей (±10 строк) |
| колесо мыши над деталями  | прокрутка тела Логов/Raw (±3 строки)                 |
| `a`                       | открыть action picker для выбранного сервиса         |
| `y`/`Enter`, `n`/`Esc`    | подтвердить / отменить открытый запрос (destructive-подтверждение или sudo-повтор) |
| `/` или `f`               | открыть фильтр меню (regex, живой ввод; `Esc` отменяет) |
| `d`                       | переключить скоуп домена: только user ↔ user + system |
| `s` / `S`                 | сменить ключ сортировки / направление               |
| `L`                       | открыть запрос пути plist для load (bootstrap)       |
| `m`                       | захватить / отпустить мышь (по умолчанию выкл → drag выделяет текст) |
| `r`                       | ручное обновление                                   |
| `q`, `Ctrl-C`             | сохранить сессию и выйти                             |

**Выделение и копирование текста** работает из коробки: мышь по умолчанию не
захвачена, поэтому обычный drag выделяет текст, а ⌘C копирует — как в текстовом
редакторе. Нажмите `m`, чтобы **захватить** мышь для кликов внутри приложения
(строки, вкладки, кнопки), прокрутки колесом и перетаскивания разделителя;
нажмите `m` снова, чтобы вернуть её к выделению текста.

Внутри action picker: `s` Start, `r` Restart, `k` Stop, `e` Enable, `d`
Disable, `u` Unload; `↑`/`↓` двигают подсветку, `Enter` выбирает, `Esc`
отменяет. Строка статуса показывает эти клавиши на каждой кнопке
(`a→[s Start]…`).

С захваченной мышью (нажмите `m`) можно кликать по строкам меню, вкладкам
деталей и кнопкам действий в строке статуса вместо клавиш, прокручивать колесом
и **тянуть разделитель** между меню и панелью деталей для изменения размеров
(панели не сожмутся ниже безопасного минимума).

## Ручной чек-лист sudo

Действия в системном домене (`system/`) требуют root; действия в
пользовательском домене (`gui/$UID`) — никогда. Этот путь всегда идёт через
настоящий `sudo` на вашем tty, поэтому проверяется вручную, а не автотестом:

- [ ] Выберите одноразовый `system`-демон, откройте action picker (`a`) и
      запустите действие (например, Start).
- [ ] Убедитесь, что best-effort-попытка падает именно с **permission**-ошибкой
      (не generic), и строка статуса предлагает **«Retry with sudo»**.
- [ ] Подтвердите повтор (`y`/`Enter`): TUI приостанавливается, `sudo`
      запрашивает пароль на настоящем терминале, и после верного пароля TUI
      возобновляется, действие проходит, список/детали обновляются.
- [ ] **Отмена:** снова откройте «Retry with sudo» и отмените (`n`/`Esc`) —
      убедитесь, что `pendingSudo` сбрасывается, статус возвращается к норме, и
      состояние не теряется (выбор/фильтр/прокрутка на месте).
- [ ] **Неверный пароль:** повторите с sudo и введите неверный пароль (или
      дайте `sudo` упасть) — убедитесь, что `pendingSudo` сбрасывается,
      показывается stderr ошибки, и новый запрос не появляется.
- [ ] **Ctrl-C на запросе:** повторите с sudo и нажмите Ctrl-C на запросе
      пароля — убедитесь, что TUI чисто возобновляется (заходит обратно в
      alt-screen), приложение **не** выходит, `pendingSudo` сбрасывается, и
      остальное состояние не меняется.

Примечание: sudo-повтор предлагают только явные **действия**. Инспекция
`system`-сервиса, который вы не можете прочитать, показывает немодальное
«requires sudo to inspect — run launchdeck with sudo» в панели деталей;
перечисление системных сервисов без root просто их опускает (запустите всё
приложение под `sudo`, чтобы их видеть).
